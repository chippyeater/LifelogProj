# 任务

请基于单个实体的局部视觉证据，生成适合后续回忆任务使用的单点细节题。

# 输入

下面是当前实体的 JSON。系统还会同时提供该实体对应的证据帧图片组，请结合 JSON 和图片一起生成细节题：

```json
{{ entity_json }}
```

活动主题：`{{ activity_name }}`

# 输出结构

只输出 JSON，顶层只包含 `detail_items`。

每个细节项必须包含：
- `question`
- `correct_answer`
- `distractors`

# 规则

1. 每个实体只生成 1 个单点细节题，不要同时问多个细节点。
2. 题目必须能从当前提供的证据帧直接验证，不能依赖常识、推测或完整事件过程。
3. 禁止生成数量题。当前阶段只有局部证据帧，无法可靠判断用户最终与同类物品发生了多少次交互，也无法确认最终数量。
4. 优先选择稳定、直接可见、单帧可验证的细节点，例如颜色、外形、包装形式、明显可见的品牌或标识、明显可见的位置关系。
5. 如果品牌、文字、型号、口味等信息在证据帧里看不清或无法确认，不要生成这类题。
6. `correct_answer` 必须具体、简洁，不要写解释句。
7. `distractors` 必须与正确答案同类型，固定为 1 个，且都是合理但错误的混淆项。
8. 不要在题干里直接泄露正确答案。
9. 如果该实体缺少足够清晰的视觉细节，不要编造复杂问题，应选择最稳定、最容易验证的细节点。

# 示例

```json
{
  "detail_items": [
    {
      "question": "这盒牛奶的包装主色是什么？",
      "correct_answer": "白色",
      "distractors": ["红色"]
    }
  ]
}
```

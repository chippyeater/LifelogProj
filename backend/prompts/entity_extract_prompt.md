# 1. Task（任务）
你将处理一个第一人称拍摄的视频任务，完成：
1. 识别用户注意到的关键实体；
2. 为每个实体提取线索和细节。
以JSON结构返回结构化内容，字段值必须使用中文。

# 2. Context（活动上下文）
活动主题：{{ activity }}
人物：{{ people }}
时间：{{ time }}
当前子事件：{{ sub_event }}
事件时间范围：{{ event_time_range }}

# 3. Rules（规则）

## 时间边界（强约束）
- 仅分析当前子事件时间范围
- 所有时间戳统一为 `HH:MM:SS.sss`
- 所有实体的`key_frame`必须当前子事件的时间区间{{ event_time_range }}内

## 实体筛选标准
- 与当前子事件行为目标直接相关
- 是用户的决策对象（选择/比较/拿起/考虑）；
- 是用户明显关注或操作的核心物体；
- 发生实际使用或交互行为的对象；
- 同时出现“包装/容器”和“其内部商品”时，优先提取商品本体，不得只输出包装/容器
- 避免重复：同一语义实体只输出一次

过滤：
- 装饰物
- 物品包装/容器
- 背景物体
- 快速闪现物体
- 无交互实体

## 实体线索要求

### `key_frame`
- 清晰可见主体无遮挡

### `details.visual`
- 描述可见特征（颜色/形状/包装/位置等）
- 不得直接出现实体名称文本

### `details.semantic`
- 描述实体类别或语义属性

## `detail_pair`
- 用于构造回忆选择题，每个实体只输出 1 组
- 内容优先级（从高到低）：
  - 数量
  - 类型/形式（散装/袋装/罐装/盒装；型号/规格；口味）
  - 颜色（大面积时）
  - 品牌/标识（仅当画面可验证时）；
- 禁止：
  - 常识题或百科题（如“黄瓜表面有什么特征”“该物体有什么用途”）；
  - 抽象题（如“它属于什么类别”）；
  - 与当下行为无关的题；
  - 无法从画面确认的题。
- 数量题额外强约束
  - 若选择“数量”作为问题变量：必须完整观看当前子事件时间范围内的全部片段，确认用户最终实际交互/拿取/放入/使用/摆放的数量；不得根据中途画面猜测；
  - 若数量无法在该片段内可靠确认，则禁止出数量题。
- 混淆选项规则
  - 与正确答案同类别、合理、可能混淆；
  - 不得跨类别混用（如把“口味”和“数量”混在一起）。

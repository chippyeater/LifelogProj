# 1. Task（任务）
你将处理一个第一人称拍摄的视频任务，完成：
1. 识别视频的主要活动主题；
2. 在活动主题下划分子事件；
3. 每个事件提供1条关键帧线索（视觉 + 语义）；
4. 提取能代表整体活动主题的一条活动层视觉线索。
以JSON结构返回结构化内容，字段值必须使用中文。

# 2. Context（活动上下文）
活动主题：{{ activity }}
人物：{{ people }}
时间：{{ time }}
地点：{{ location }}
视频总时长：{{ video_length }}

# 3. Rules（规则）

## `activity_description`（活动内容概括）
  - 简要概括，不超过10个字；
  - 不列举具体物品。

## 子事件边界划分标准

### 划分依据
- 子事件粒度应为“同一空间区域 + 同一购买目标”的连续行为；
- 不得按单个具体商品划分事件；
- 最后一个事件`end_time`必须等于或接近`video_length`；后半段仍有明显行为变化必须继续划分；不得提前终止。

### 合并规则
- 无明确目标的连续动作应合并
- 非关键行为（如短暂看手机、路过某物）应并入更大行为单位
- 遮挡严重、无可识别信息的片段不单独成事件
- 同一区域内对多个同类商品的浏览、对比、拿取，应合并为同一事件；

## 子事件字段要求
### `name`（事件名）
- 动宾结构
- 省略第一人称
- 5～12字
- 不含时间词
- 不得出现具体品牌或具体商品名称；
- 示例：挑选蔬菜、挑选零食、进入超市、结账付款

### `description`（事件描述）
长度建议：30～60字，1～2句，包含：
- 场景位置
- 行为目标
- 关键物体或人物
- 交互动作
- 可见细节（颜色 / 包装 / 数量 / 标识等）

禁止：
- 列举无关物品
- 抽象总结（如“进行购物活动”）

示例：
- “走到冷柜旁，我们打开柜门，拿了一盒香蕉味的牛奶，还为自己选了一瓶瓶装的芬达。”
- “我们逛到了零食区，我们拿了红色包装的泡面、芝麻味的饼干和两包瓜子。”

## `scene_clue`（线索提取标准）

### `frame`
- 格式：`HH:MM:SS.sss`
- 必须在事件时间区间内
- 画面清晰无遮挡

### `description`（可验证线索）
- 1～2句，30～60字优先；
线索应从以下方面提取：
- **环境颜色特征**：例如，“冷柜内的瓶装饮料标签上有鲜艳的红色字体”；
- **环境声音线索**：例如，“背景中能听到咖啡机运作的声音”；
- **感官细节**：例如，“空气中弥漫着咖啡的香气，温度稍微偏高”；
- **物体或场景的独特细节**：例如，“左侧墙上有醒目的蓝色‘出口’标识”。

### `activity_visual_clue`（活动层视觉线索）
- 结构同`scene_clues`
- 代表整体活动
- 不可重复单一子事件内容
